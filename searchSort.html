Big O notation

  • For time complexity used to measure the times an element is accessed in an array
  • Space complexity used a similar notation for the amount of elements stored in memory
  • For example if we access all the indexes of an array, we have a O(n) time complexity
  • Then, if we copy those indexes to another array, we have a O(n) space complexity
  • Big O notation allows to carefully examine alogrithims and find alternatives
  • For Sake of Simplicity, when seeing log it is refering to log2

Searching

Finding the index of an element requires O(n) time if an array is unsorted as we need to traverse. However, if sorted we can use Binary sort to more quickly find an element

Binary Search

  • The Precondition is that the array is sorted
  • Plays higher or lower until finding the target index
  • Time-Complexity Olog(n)
  • Java has a built-in function: Arrays.binarySearch()

Copy

public static int bSearch(int[] arr, int left, int right, int x){
    if (right >= left){
        int mid = (left + right) / 2;
        if (arr[mid] == x){
            return mid;
        }
        else if (arr[mid] > x){
        return bSearch(arr, left, mid - 1, x);
        }
        else{
            return bSearch(arr, mid + 1, right, x);
        }
    }
    return -1;
}
